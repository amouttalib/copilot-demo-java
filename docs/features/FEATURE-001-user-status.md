# FEATURE-001 – Statut utilisateur

---

## 1. Description fonctionnelle (HUMAIN)

Ajouter un statut à l’utilisateur :

- ACTIVE
- INACTIVE
- SUSPENDED

Règles métier :
- Le statut par défaut est ACTIVE
- Un utilisateur INACTIVE n’est pas retourné par l’API
- Un utilisateur SUSPENDED n’a pas accès aux fonctionnalités sensibles
